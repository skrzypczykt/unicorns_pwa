# Archiwum Dokumentacji

⚠️ **UWAGA:** Dokumenty w tym folderze mogą zawierać nieaktualne informacje.

Zostały przeniesione tutaj jako archiwum historyczne - mogą być pomocne do zrozumienia historii projektu i decyzji architektonicznych, ale nie należy ich traktować jako aktualnej dokumentacji.

## Lista Dokumentów

- **AUTOMATIC_LOGOUT_IMPLEMENTATION.md** - Plan implementacji auto-logoutu (prawdopodobnie już zaimplementowane)
- **CASE_STUDY_REREGISTRATION.md** - Case study problemu rejestracji (historyczne)
- **CSP_IMPROVEMENTS.md** - Propozycje ulepszeń Content Security Policy
- **FIX_REDIRECT_URL.md** - Fix dla redirect URL (prawdopodobnie naprawione)
- **LOCAL_TESTING_PLAN.md** - Plan testów lokalnych (może być przestarzały)
- **PAYMENT_TRACKING_TODO.md** - TODO dla payment tracking (sprawdź czy ukończone)
- **REGISTRATION_READY.md** - Notatki o gotowości systemu rejestracji
- **SECURITY_IMPROVEMENTS.md** - Propozycje bezpieczeństwa (niektóre mogły być wdrożone)
- **test-member-zone.md** - Testy strefy członka
- **update_seed_uuids.md** - Instrukcje update'u UUID w seed.sql

## Przed użyciem

Jeśli chcesz wykorzystać informacje z tych dokumentów:
1. Sprawdź aktualny kod źródłowy czy feature już istnieje
2. Porównaj z aktualną dokumentacją w `/docs/guides` i `/docs/deployment`
3. W razie wątpliwości - stwórz nowy plan zamiast opierać się na archiwum

## Sugestie

Jeśli znalazłeś tutaj dokument, który:
- Jest nadal aktualny → przenieś go do właściwego folderu (`/docs/guides` lub `/docs/deployment`)
- Został w pełni zaimplementowany → możesz go usunąć
- Ma historyczną wartość → zostaw tutaj
